## Appendix A: Glossary of Terms

| Term | Definition |
|------|------------|
| Allocation | The assignment of a specific amount from the organizational budget to a particular event. |
| Audit Trail | An immutable, chronological log of all critical actions performed in the system, including who performed them and when. |
| bcrypt | A password hashing algorithm used to securely store user passwords. It applies a salt and multiple rounds of hashing to prevent brute-force attacks. |
| Checklist | A task list associated with a specific event, used to track preparation progress and ensure event readiness. |
| Checklist Template | A reusable set of checklist items created by an Admin that can be applied to any event to generate a starting checklist. |
| Cookie | A small piece of data stored in the user's browser, used by Structura to maintain the user's session after login. |
| Deployment | The process of publishing the application to a live server so it is accessible to users over the internet. |
| Document | A file uploaded to the system and linked to an event. Supported types include permits, contracts, promotional materials, receipts, and financial records. |
| Event | A proposed or approved activity organized by a student organization, managed through the platform from proposal to completion. |
| Event Status | The current lifecycle stage of an event: Proposed, Approved, Completed, or Cancelled. |
| Expenditure | A recorded expense charged against an event's allocated budget, supported by a financial document. |
| Free Tier | A no-cost usage plan offered by cloud services such as Vercel and Supabase, with limited resources suitable for small-scale or development use. |
| iron-session | A stateless session library for Next.js that stores encrypted session data directly in browser cookies without requiring server-side session storage. |
| Middleware | Code that runs between a request and a response, used in Structura to enforce authentication and role-based access on protected routes. |
| Migration | A versioned script that creates or modifies database tables and schemas in a controlled and repeatable way. |
| Officer | A user role in Structura representing an administrative office staff member with elevated permissions over student submissions. |
| Organizer | A user role in Structura representing a student organization member who can create and manage event proposals. |
| Organization | The student organization entity registered in the system, associated with users, events, and a shared budget. |
| Realtime | A feature of Supabase that pushes live database change events to connected clients via WebSocket, enabling the UI to update without a page refresh. |
| Role | A classification assigned to each user that determines what features and actions they can access within the platform. |
| Salt | A random value added to a password before hashing to ensure that identical passwords produce different hash outputs. |
| Server Action | A Next.js feature that allows server-side functions to be called directly from client components without writing a separate API endpoint. |
| Session | A temporary authenticated state stored in an encrypted browser cookie that identifies the logged-in user across requests. |
| Serverless Function | A backend function hosted on Vercel that runs on-demand in response to HTTP requests, without a persistently running server. |
| Supabase | A managed backend platform providing PostgreSQL database, file storage, and real-time capabilities used as the data layer for Structura. |
| Vercel | A cloud deployment platform used to host the Structura Next.js application and its serverless API routes. |
| WebSocket | A communication protocol that enables a persistent, two-way connection between the browser and server, used by Supabase Realtime for live updates. |
